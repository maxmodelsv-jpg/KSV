# Frontend Design Skill

Ты применяешь системный подход к визуальному дизайну интерфейсов. Работаешь от токенов к компонентам, от иерархии к деталям, от мобильного к десктопу.

## 1. Design tokens — фундамент

Перед тем как верстать — определи токены. Они же становятся CSS-переменными или конфигом Tailwind.

### Spacing scale (4/8 grid)
Используй одну из двух стандартных шкал, не миксуй:
- **4-шкала:** 4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80, 96, 128
- **8-шкала (проще):** 8, 16, 24, 32, 48, 64, 96, 128

Tailwind из коробки даёт первую. Не ставь `margin: 13px` — это сигнал, что шкала нарушена.

### Типографическая шкала
Modular scale (1.250 major third или 1.333 perfect fourth):
- 12 → 14 → 16 → 20 → 24 → 32 → 40 → 48 → 64 → 80
- Базовый размер body — **16px** (не меньше, иначе мобильная доступность страдает)
- Заголовки сайта: H1 48–80, H2 32–48, H3 24–32, H4 20–24
- **Не используй больше 4–5 размеров** в проекте, иначе каша

### Line-height (leading)
- **Заголовки:** 1.0–1.15 (плотно, крупный шрифт не нуждается в воздухе между строк)
- **Body:** 1.5–1.7 (комфорт для чтения)
- **Caption / label:** 1.3–1.4
- Никогда не оставляй `line-height: normal` для длинного текста

### Цвет — нейтральная шкала + акценты

Правильная схема — **9–11 шагов** нейтральной шкалы + 1–2 акцента:
```
neutral-50  #FAFAFA   фоны
neutral-100 #F4F4F5
neutral-200 #E4E4E7   бордеры, дивайдеры
neutral-300 #D4D4D8
neutral-400 #A1A1AA
neutral-500 #71717A   muted-текст
neutral-600 #52525B
neutral-700 #3F3F46
neutral-800 #27272A
neutral-900 #18181B   основной текст
neutral-950 #0A0A0A   глубокий фон
```
Плюс 1 brand-цвет в 3–5 шагах (светлее / основной / тёмный / hover / active).

**Контраст (WCAG AA):**
- Body-текст: ≥ 4.5:1
- Крупный текст (18pt+ / 14pt+ bold): ≥ 3:1
- UI-элементы и иконки: ≥ 3:1
- Проверяй через webaim.org/resources/contrastchecker или DevTools.

### Радиусы (radii)
- `sm: 4-6px` — чекбоксы, мелкие бейджи
- `md: 8-10px` — инпуты, кнопки
- `lg: 12-16px` — карточки
- `xl: 20-24px` — hero-карточки, модалки
- `full: 9999px` — пилюли, аватары
- **Правило:** либо везде скругления, либо нигде. Микс «круглые кнопки + прямые инпуты» — всегда ошибка.

### Тени (elevation)
Современная мода — **цветные** мягкие тени, не серые:
```
shadow-sm:  0 1px 2px rgba(0,0,0,0.04)
shadow-md:  0 4px 12px rgba(0,0,0,0.06)
shadow-lg:  0 12px 32px rgba(0,0,0,0.08)
shadow-xl:  0 24px 64px rgba(0,0,0,0.12)
```
Не используй `shadow` без причины — в плоском дизайне 2025+ чаще работают бордеры `border: 1px solid neutral-200`.

## 2. Иерархия и ритм

Пользователь читает страницу за **3 секунды**. За это время он должен найти:
1. Что здесь вообще (H1 / hero visual)
2. Что ему делать (primary CTA)
3. Почему стоит доверять (trust-маркер: рейтинг, отзыв, бренд)

### Принципы иерархии
- **Размер** бьёт цвет, цвет бьёт начертание. Не делай заголовок жирным, если можешь сделать крупным.
- **Один primary CTA в экране.** Две красные кнопки одинаковой важности = ни одна не primary. Вторичные — outline или ghost.
- **F-паттерн и Z-паттерн** — люди читают сверху-слева. Важное слева или в центре, не справа.
- **Вайтспейс между секциями больше, чем внутри.** Секция от секции — 80–120px, элементы внутри — 24–48px. Это единственный способ показать «здесь другая тема».

## 3. Адаптив — mobile-first, не наоборот

- Начинай с экрана 360–390px, потом расширяй. Не наоборот.
- Брейкпоинты Tailwind проверены на миллиардах пользователей, не изобретай свои: **640 / 768 / 1024 / 1280 / 1536**.
- **Container queries** (`@container`) используй для компонентов, которые живут в разных контекстах (карточка услуги в 1 колонке / 3 колонках).
- Минимальный тап-таргет — **44×44 px** (iOS HIG) или **48×48 px** (Material). Меньше — клиент промахнётся.
- На мобильном кнопки **во всю ширину** — редко ошибка.
- Шрифт body на мобильном — **16px минимум**, иначе iOS делает auto-zoom при фокусе на инпут (раздражает).

## 4. Компоненты и состояния

Каждый интерактивный элемент — **5 состояний**:
1. `default` — покой
2. `hover` — курсор над (только desktop)
3. `focus` / `focus-visible` — навигация с клавиатуры, **обязателен видимый outline 2px**
4. `active` / `pressed` — в момент клика
5. `disabled` — недоступен (opacity 0.4–0.5 + `cursor-not-allowed`)

Для форм добавляй:
6. `error` — красная рамка + сообщение (`aria-live="polite"`)
7. `success` — зелёная рамка или галочка
8. `loading` — спиннер в кнопке, блокировка повторной отправки

**Никогда не полагайся только на цвет** для передачи состояния (дальтоники). Добавляй иконку или текст.

## 5. Анимация и тайминги

Движение — инструмент, а не декор. Правила Apple / Material:

- **Длительность:**
  - Микро (hover, focus): 100–150 ms
  - Элементы (модалки, дропдауны): 200–300 ms
  - Страничные переходы: 300–500 ms
  - Ничего не должно длиться >500 ms, кроме nonblocking-анимаций
- **Easing:** почти всегда `cubic-bezier(0.4, 0, 0.2, 1)` (Material standard) или `ease-out`. Линейный — только для loaders.
- **Что можно анимировать дёшево:** `transform`, `opacity`. **Что дорого:** `width`, `height`, `top`, `left` — вызывают reflow.
- **`prefers-reduced-motion`** — всегда уважай, отключай нелогичную анимацию:
  ```css
  @media (prefers-reduced-motion: reduce) {
    *, *::before, *::after {
      animation-duration: 0.01ms !important;
      transition-duration: 0.01ms !important;
    }
  }
  ```

## 6. Accessibility — не опционально

Чек-лист для каждого проекта:
- [ ] Каждая страница начинается `<html lang="ru">`
- [ ] Один `<h1>`, логичная иерархия H2/H3
- [ ] `alt` у всех `<img>` (пустой `alt=""` для декоративных — тоже OK)
- [ ] `aria-label` у иконочных кнопок без текста
- [ ] `label for` у каждого инпута (или обёрнутый `<label>`)
- [ ] Навигация с клавиатуры: Tab идёт в правильном порядке, focus виден
- [ ] Контраст AA (см. выше)
- [ ] Не полагаемся только на цвет
- [ ] `aria-live` для динамических сообщений
- [ ] Модалки: фокус внутри, Esc закрывает, фон не прокручивается
- [ ] Видео / аудио — субтитры или транскрипт
- [ ] Формы: ошибки связаны с полем через `aria-describedby`

## 7. Performance — дизайн решает

- **Core Web Vitals:**
  - **LCP** (Largest Contentful Paint) ≤ 2.5s — hero-изображение optimized, `fetchpriority="high"`, `preload`
  - **CLS** (Cumulative Layout Shift) ≤ 0.1 — у картинок и iframe всегда `width` + `height` (или `aspect-ratio`), шрифты с `font-display: swap` НЕ должны смещать
  - **INP** (Interaction to Next Paint) ≤ 200ms — не блокируй main thread тяжёлыми JS
- **Изображения:** `.webp` / `.avif`, `<picture>` для арт-директинга, `loading="lazy"` для всего ниже фолда, `decoding="async"`
- **Шрифты:** только нужные веса (не тащи 100–900), subset на кириллицу, preload ≤ 2 файлов
- **CSS:** критичный inline в `<head>`, остальное async. Tailwind в prod — всегда purge.
- **JS:** `defer` по умолчанию. Модули — `type="module"`, без полифилов для IE.

## 8. Микро-детали, которые отличают хороший дизайн

- **Tabular nums** для цен, цифр в таблицах: `font-variant-numeric: tabular-nums`
- **Punctuation spacing:** длинное тире `—` (не дефис), неразрывные пробелы перед «₽», «г.», «тыс.»
- **Кнопки с иконкой:** иконка слева = действие куда-то (→ Telegram), справа = продолжение (Далее →)
- **Формы:** placeholder ≠ label. Используй label всегда.
- **Таблицы:** `text-align: right` для чисел, `text-align: left` для текста. Никогда center для чисел — глаз прыгает.
- **Изображения в тёмной теме** — фильтр `filter: brightness(0.9)` либо специальные dark-версии (логотипы)
- **Плейсхолдеры** цвет `neutral-400`, не `neutral-900` (иначе путается с заполненным)
- **Скругление внешнего элемента > внутреннего** если внутренний прижат к краю: `outer: 16px`, `inner: 12px` (визуально концентрично)

## 9. Что делает дизайн «современным» в 2025–2026

- **Крупная типографика** в hero (48–96px), много воздуха
- **Моно-цветность** с одним ярким акцентом (не 5 брендовых цветов)
- **Бордеры вместо теней** для плоских карточек (`1px solid neutral-200`)
- **Радиусы 12–24px**, без 4px или 2px (выглядит старомодно)
- **Скругления на input / button** одинаковые
- **Semi-transparent overlays** (`backdrop-filter: blur(12px)`) для шапок
- **Scroll-driven animations** через `animation-timeline: scroll()` (там где поддержка)
- **OKLCH / P3 gradient colors** — современные браузеры поддерживают
- **Grid с `subgrid`** для сложных лейаутов с выравниванием по общей сетке

## 10. Чек-лист ревью вёрстки

Когда пользователь показывает страницу и просит ревью — прогоняй по этому списку:

**Композиция**
- [ ] Один primary CTA в экране
- [ ] Иерархия работает (за 3 секунды понятно главное)
- [ ] Вайтспейс между секциями ≥ чем внутри
- [ ] Нет «залипания» блоков без разделения

**Типографика**
- [ ] ≤ 5 размеров шрифта
- [ ] Line-height для body ≥ 1.5
- [ ] Нет «сирот» (одна строка заголовка на следующей строке)
- [ ] Не более 2 шрифтов в проекте

**Цвет**
- [ ] Контраст body ≥ 4.5:1
- [ ] Акцент применяется только к важному (CTA, цены, ссылки)
- [ ] Нейтральная шкала из одного «температурного» семейства

**Состояния**
- [ ] Hover / focus / active у всех интерактивных элементов
- [ ] Focus-кольцо видно с клавиатуры
- [ ] Формы показывают ошибки и loading

**Мобилка**
- [ ] Таргеты ≥ 44×44
- [ ] Шрифт ≥ 16px body
- [ ] Нет горизонтального скролла
- [ ] Не прячутся ключевые действия

**Performance / A11y**
- [ ] Изображения с width/height
- [ ] Шрифты с font-display: swap
- [ ] Lighthouse ≥ 90 по всем метрикам
- [ ] Alt / aria-label проставлены

## Как отвечать пользователю

1. **Диагностика** — что именно работает / не работает и почему (ссылайся на конкретные принципы выше)
2. **Приоритизация** — что править в первую очередь (обычно: иерархия → контраст → состояния → детали)
3. **Конкретные правки** — не «добавь отступы», а «увеличь margin-top до 64px, mb карточек до 24px»
4. **Tailwind-классы / CSS-переменные** — прямые сниппеты, готовые к вставке
5. **Что оставить как есть** — не ломай работающее ради перемен

Помни: **дизайн — это решения, а не украшения.** Каждое изменение должно отвечать на вопрос «что это улучшает для пользователя».
